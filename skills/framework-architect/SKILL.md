# Framework Architect - 架構設計 AI 導入規格產生器

> 🏗️ 為您的架構/系統設計完整的 AI 導入規格，讓 AI 助手快速理解並幫您整合到其他專案

## 功能說明

此 skill 會引導您描述一個架構或系統，並自動產生：
1. **完整規格文檔（YAML 格式）**：包含系統概述、核心架構、實作細節、使用指南
2. **AI 整合指令模板**：可直接複製給 AI 使用的整合 prompt
3. **快速開始文檔（README）**：給開發者或 AI 閱讀的完整使用指南

## 使用場景

✅ **適合使用此 skill：**
- 您開發了一套可重用的系統架構（如 API 管理、Prompt 管理、資料處理流程）
- 希望將此架構快速應用到其他專案
- 需要讓 AI 理解並幫您整合
- 想要完整文檔化您的系統設計

❌ **不適合：**
- 簡單的單一功能（不需要完整架構）
- 一次性的專案特定邏輯

## 執行方式

當使用者呼叫此 skill 時，請按照以下步驟引導：

### 第 1 步：收集基本資訊

請詢問使用者以下問題：

```
【基本資訊】
1. 這個架構/系統的名稱是什麼？
2. 它解決了什麼問題？（3-5 句話描述）
3. 核心功能有哪些？（列出 3-5 個主要功能）
4. 使用的技術棧是什麼？（語言、框架、套件）
```

### 第 2 步：收集架構資訊

```
【架構設計】
5. 核心設計原則是什麼？（例如：單一來源原則、循環輪替、集中式管理）
6. 主要組件有哪些？（列出 2-5 個核心組件及其職責）
7. 資料流向是什麼？（簡單描述數據如何在組件間流動）
8. 關鍵的類別或模組有哪些？（列出主要的 class/module 及其方法）
```

### 第 3 步：收集實作細節

```
【實作細節】
9. 目錄結構是什麼？（列出主要檔案和資料夾）
10. 有哪些核心程式碼檔案？（列出 3-5 個關鍵檔案及其用途）
11. 設定檔是什麼格式？（YAML、JSON、.env 等）
12. 有哪些環境變數或設定參數？
```

### 第 4 步：收集整合資訊

```
【整合指南】
13. 整合步驟有哪些？（複製檔案、修改設定、更新程式碼等）

15. 有哪些常見問題或陷阱？（已知問題和解決方案）
```

```python
# 偽代碼
def upload_to_github(framework_dir, repo_url, branch, create_pr):
    """
    上傳架構到 GitHub 架構庫
    """
    # 1. 檢查 repo 是否存在
    if not repo_exists(repo_url):
        print(f"⚠️ Repository {repo_url} 不存在")
        if user_confirms("是否建立新的 repository？"):
            create_github_repo(repo_url)
        else:
            return

    # 2. Clone 或更新本地 repo
    local_repo = clone_or_pull(repo_url, branch)

    # 3. 複製架構檔案
    target_dir = f"{local_repo}/frameworks/{framework_name}/"
    copy_framework_files(framework_dir, target_dir)

    # 4. 更新架構索引
    update_framework_index(local_repo, framework_name)

    # 5. Commit 和 Push
    git_add_commit_push(local_repo, framework_name, branch)

    # 6. 建立 PR（如果需要）
    if create_pr:
        pr_url = create_pull_request(repo_url, branch, framework_name)
        print(f"✅ Pull Request 已建立：{pr_url}")
    else:
        print(f"✅ 已 push 到 {repo_url}")

    # 7. 返回 GitHub URL
    github_url = f"{repo_url}/tree/{branch}/frameworks/{framework_name}"
    return github_url
```
